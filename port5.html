Overview.

While I am not heavily dependent on my cyber security skills, I am fairly knowledgeable on how to spot certain web vulnerabilities, and even how to mitigate them. With this in mind, here is an example of a project that I completed in school involving pen testing and other security protocols.

There were several vulnerabilities in the assigned website that allowed attackers to gain access to the site without authorization. Not only this but, attackers were able to login as random users, specific users, and even register as new users with an unlimited amount of funds without a credit card information required. In the first example above, I was able to exploit the vulnerability in the LoginAction page and perform a SQL injection attack in the login screen in order to login as a random user. Using (“‘ or ‘a=’a ”) as the password, I was able to bypass the user authentication process and login to the website with out and issue, because the SQL injection I used was making sure that no matter what user I type in the text field the password will always be true, allowing me to login as the first user in the database, which was the user "johnDoe" in this case.

In the second example, I demonstrate a XSS vulnerability found on the Tunestore website, this time located on the comment’s page textbox area. Exploiting this vulnerability I was able to inject the website with a script that displays an alert as soon as another user clicks on the comments page. I started by logging in as a specific user already registered to the site. Then navigated over to the comments page of one of the CDs and typed the following script on the comments box (“<script> alert(“You have been haceked!! Click her to recover account”); </script>”) and clicked on submit, and successfully inject my code in to the website. This can be very malicious and attackers can use this type of vulnerability to phish for user data.
